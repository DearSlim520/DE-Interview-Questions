## Q1. DQC 六大维度

> 📌 **频率**: 2024-2025 · ★★☆

### 🎯 六大维度一览

| 维度 | 英文 | 校验示例 |
|------|------|----------|
| **完**整性 | Completeness | `order_id IS NULL` 占比 < 0.01% |
| **准**确性 | Accuracy | 值域、枚举、与上游对账 |
| **一**致性 | Consistency | 跨表主外键、跨系统口径一致 |
| **唯**一性 | Uniqueness | `COUNT(DISTINCT pk) = COUNT(*)` |
| **及**时性 | Timeliness | 产出时间 ≤ SLA（如 09:00 前） |
| **有**效性 | Validity | 格式/类型合规 |

### 📝 实战示例：`dwd.order` 表

```yaml
rules:
  完整性:   order_id IS NULL ratio < 0.01%
  唯一性:   COUNT(DISTINCT order_id) = COUNT(*)
  同环比:   今日记录数 vs 7日均值 ±30%
  及时性:   产出时间 < 09:00，否则告警
  一致性:   SUM(amount) vs 上游 ODS 偏差 < 0.1%
```

### ⚡ 告警分级

| 级别 | 条件 | 处理 |
|------|------|------|
| 🔴 阻塞 | 主键重复 / NULL 率爆表 | 阻断下游任务 |
| 🟡 弱告警 | 记录数波动超阈值 | 通知 + 人工确认 |

### 🧠 记忆锚点

```
6 维度 = 完 · 准 · 一 · 唯 · 及 · 有
```

## Q2. 数据血缘系统设计

> 📌 **频率**: 2024-2025 上升题 · ★★☆

### 🎯 四层架构

```
┌──────────────────────────────────────────────────────┐
│  ④ 服务层   API 查询上下游 · 影响分析 · 可视化       │
├──────────────────────────────────────────────────────┤
│  ③ 存储层   Neo4j / JanusGraph                       │
│             节点: 表 · 字段 · 任务                    │
│             边:   产出 · 消费 · 字段映射              │
├──────────────────────────────────────────────────────┤
│  ② 解析层   Apache Calcite / Antlr + 自研            │
│             提取 input/output 表 + 字段映射           │
├──────────────────────────────────────────────────────┤
│  ① 采集层   Hive Hook · Spark Listener · Airflow CB  │
│             拿到执行的 SQL / DAG                      │
└──────────────────────────────────────────────────────┘
```

### 📝 解析示例

```sql
INSERT INTO dwd.user_order
SELECT a.uid, b.amount
FROM ods.user a JOIN ods.order b ON a.uid = b.uid
```

**表级血缘**：
```
ods.user ──┐
            ├──► dwd.user_order
ods.order ─┘
```

**字段级血缘**：
```
dwd.user_order.uid    ← ods.user.uid
dwd.user_order.amount ← ods.order.amount
```

### ✅ 应用场景

| 场景 | 说明 |
|------|------|
| 影响分析 | `ods.user` 字段改类型 → 一键查所有下游受影响任务/报表 |
| 数据溯源 | 报表数据异常 → 逆向追踪到源头表 |
| 权限治理 | 敏感字段血缘链路 → 自动继承脱敏策略 |

### 🧠 记忆锚点

```
采集 → 解析(Calcite) → 图存储(Neo4j) → 影响分析
```
